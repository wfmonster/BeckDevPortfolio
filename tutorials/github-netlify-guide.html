Overview

This guide walks you through setting up GitHub for version control and Netlify for free web hosting. By the end, you'll have your own website live on the internet with automatic deployments.

  • Setting up a GitHub account and repository
  • Basic Git commands for version control

  • Daily workflow for managing your site

Part 1: GitHub Setup

Create a GitHub Account

1

Sign Up

Go to github.com and click "Sign up". Follow the prompts to create your account and verify your email address.

Install Git

2

macOS (with Homebrew)

brew install git

Or download from git-scm.com/downloads

Configure Git

3

Set Your Identity

Tell Git who you are (use the same email as your GitHub account):

git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

Create a Repository

4

On GitHub

Click the "+" icon in the top right ‚Üí "New repository". Name your repo (e.g., my-website), choose Public or Private, and click "Create repository". Don't initialize with a README if you're pushing existing code.

Push Your Project

5

Connect Local to GitHub

In your terminal, navigate to your project folder and run:

# Navigate to your project
cd /path/to/your/project

# Initialize git (if not already done)
git init

# Add all files
git add .

# Commit
git commit -m "Initial commit"

# Connect to GitHub (use YOUR repo URL)
git remote add origin https://github.com/USERNAME/REPO.git

# Push to GitHub
git branch -M main
git push -u origin main

Part 2: Netlify Deployment

Create a Netlify Account

1

Sign Up with GitHub

Go to netlify.com, click "Sign up", then "Sign up with GitHub". Authorize Netlify to access your GitHub account.

Deploy Your Site

2

Import from GitHub

From the Netlify dashboard, click "Add new site" ‚Üí "Import an existing project" ‚Üí "Deploy with GitHub". Select your repository.

3

Configure Build Settings

For static HTML sites:

  • Build command: Leave blank
  • Publish directory: Leave blank or enter .

Click "Deploy site"!

4

Your Site is Live!

Netlify assigns a random URL like random-name-123.netlify.app. Every push to GitHub will automatically redeploy your site in about 30 seconds.

Customize Your URL

5

Change Site Name

Go to Site settings ‚Üí Site details ‚Üí Site information ‚Üí "Change site name". Choose something memorable like your-project.netlify.app.

Part 3: Daily Workflow

Once set up, updating your site is simple:

üîÑ Update Workflow
  1. Edit files locally
  2. Run git add . && git commit -m "message"
  3. Run git push
  4. Netlify auto-deploys (~30 seconds)
  5. Changes live on the web!

Common Commands

# Check what's changed
git status

# Stage all changes
git add .

# Commit with a message
git commit -m "Describe what you changed"

# Push to GitHub (triggers Netlify deploy)
git push

# Pull latest changes (if working on multiple computers)
git pull

# See commit history
git log --oneline

Command Reference

# Clone an existing repo
git clone https://github.com/username/repo.git

# Undo uncommitted changes to a file
git checkout -- filename

# Create a new branch
git checkout -b new-feature

# Switch branches
git checkout main

# See remote URL
git remote -v

# Change remote URL
git remote set-url origin NEW_URL

Troubleshooting

"Permission denied" when pushing?

Set up SSH keys following the GitHub SSH Guide, or use HTTPS with a personal access token.

Netlify build failed?

Check the deploy log in Netlify for errors. For static HTML sites, this usually means a file path issue or missing file.

Changes not showing on live site?

Wait 30-60 seconds for the deploy to complete. Then hard refresh your browser: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows).

Git says "nothing to commit"?

You haven't made any changes since your last commit, or you need to save your files first.
