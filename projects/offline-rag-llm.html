Offline RAG LLM System

Python FastAPI Next.js LangChain Ollama ChromaDB

A fully offline AI chat application with document Q&A ‚Äî no cloud required.

Overview

This project is a complete Retrieval-Augmented Generation (RAG) system that runs entirely on local hardware. It enables private, secure AI conversations with the ability to query your own documents ‚Äî all without sending data to external servers.

Built with privacy-first principles in mind, this system is ideal for researchers, professionals handling sensitive documents, or anyone who wants the power of modern AI without cloud dependencies.

Key Features

Chat Interface & Management

  • Real-time chat interface with message threading
  • Chat history sidebar with timestamps
  • Full CRUD operations: create, rename, delete conversations
  • Auto-generated chat titles based on first message
  • Copy to clipboard functionality for all messages

RAG Pipeline (Document Q&A)

  • Document upload with async processing
  • Markdown and text file support
  • Intelligent text chunking (1000 chars, 200 overlap)
  • Vector storage with ChromaDB
  • Semantic search for relevant context retrieval
  • Source attribution in responses

Document Library

  • Drag-and-drop upload interface
  • Folder organization with custom colors
  • Tag-based categorization and filtering
  • Live markdown editor with preview
  • Chunk preview to see how documents are split

Prompt Template Library

  • Save, edit, and organize prompt templates
  • Category-based organization
  • Favorites system for quick access
  • One-click "Start Chat" with template

Additional Features

  • Dark/light theme with system detection
  • Per-chat document context selection
  • Responsive design for desktop and mobile
  • Full keyboard navigation support

Architecture

The system follows a microservices-inspired architecture with clear separation between frontend, backend API, database, and AI inference layers:

System Flow
  1. User uploads documents ‚Üí Backend processes and chunks text
  2. Chunks are embedded and stored in ChromaDB vector database
  3. User asks a question ‚Üí Semantic search retrieves relevant chunks
  4. Context + question sent to Ollama for local LLM inference
  5. Response streamed back with source attribution

Privacy & Security

This project was designed with data sovereignty in mind:

  • 100% Local: All processing happens on your machine
  • No API Keys: No OpenAI, no cloud LLM providers
  • Your Data Stays Yours: Documents never leave your network
  • Air-Gapped Ready: Can run completely offline
